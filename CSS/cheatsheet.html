From HTML Page: List of Style Properties

Color

Changes the color of a piece of text. Can also be used in the body tag to change all the text for the page. Note for all the color ones, the color can be defined by hex (#ffffff), rgb (rgb(255, 12, 72, 1)), hsl (hsl(9, 100%, 64%)), or by pre-defined color names (tomato, palegreen)

Border

Addes a border, a little rectange, around the element. You define the width of the border in pixels, the color and if it's solid that color. Also, borders exists for each direction (top, bottom, left, right).

Border-Radius

Addes rounded corners to borders

Border-Style

Defines the type of line used in the borders: solid, dotted, dashed, double, groove, ridge, inset, outset, none, hidden (default)

Border-Width

Chooses the width the a border, entering two values put its first for the top and bottom of the border, and the second value for the sides.

Border-Color

Defines the colors of the borders, can have up to 4 values for the top, right, bottom, and left sides.

Border-Image

Defines an image to use as a border instead of a line. Formatted as image url, where to cut border image, property. Round repeats the non-corner elements, stretch streches the non-corner elements (this uses round).

Padding

Defines, in pixels, the space between the text and a border element. Also, padding exists for each direction (top, bottom, left, right).

Box-Sizing

Set the box-sizing property to border-box to ensure your element stays a desired size, and padding doesn't make it take up more space.

Margin

Defines the margin outside of a border.

It effects how close it is to other elements. There are also properties for margins of each side of the border.

Auto will horizonally center the element, inherit will give it the margin of its parent element (ex. a header in a div). To center images, margin-left and margin-right need to be set to auto.

Border-Collapse

Used to prevent borders next to each other creating a double border effect (like in a table).

Outlines

Outlines are lines drawn around borders, and have the same elements as a border (style, color, width) and offset, the space between a border and an outline.

Link Colors

The color of links in different states can be changed with a:link, a:visited, a:hover, or a:active in a style sheet. This also works in buttons.

Font-Family

Defines the font used on the page or in a piece of text.

Font-Size

Changes the font size of the text. Can be defined by a size in pixels (better for single lines) or modified their base size by a percentage (font-size: 300%;). Can also be defined by viewport width (vw) which is more accomodating to more devices, or em (1 em is 16px (sta)).

Text-Align

Changes the text alignment like left, center, right, etc.

Float

Makes an image float to the left/right of text. Can also use width/height in style to size images to prevent issues with style sheets.

Background-Image

Makes the background of the page to an image, formatted like url('link').
background-color: change the base color of a webpage
background-repeat: no-repeat can be used to stop a small image from repeating along the screen.
background-attachment: fixed can be used to make sure the background image doesn't scroll.
background-size: cover or 100% 100% allow it to stretch an image to the size of the screen.
background-position: specifies the position of a backgound image if not repeating.
Note: the background property can be used to change all of these in one declaration (ex: background: #ffff00 url("img_link") no-repeat right top;). Additional note, you can also have multiple background-images, separated by commas (i.e (url(img 1), url(img 2)) (right bottom, left top)).

@Media

Not a tag in style, but you can use @media screen to check the resolution of the screen to do different things with style similar to picture element stuff.

Canvas

A canvas element is a container in which to draw graphics on the fly with JavaScript. An example of how to declare a canvas is as follows: <canvas id='name' width='x' height ='y' style='border:1px solid #000000;'>.

Drawing on a Canvas

To draw on the canvas, Javascript code has to be written. The following code, for example, would draw a line diagionally across the canvas.

        
            var canvas=document.getElementById('myCanvas');
            var pen=canvas.getContext('2d');
            pen.moveto(0,0);
            pen.lineto(100,100);
            pen.stroke();

Moveto and Lineto

In the canvas, the moveTo method moves the pen object to a specific point, and the lineTo method makes a line from where the pen is to a point specific point.

BeginPath, Arc, and Stroke

In the canvas, the beginPath method is used to reset/set a path for the pen object, and arc defines a arc to be drawn on the canvas that is confirmed with the stroke method. In the arc method, the following values are taken: pen.arc(x,y,radius,start angle, end angle) (the angles are in radians, 0 to 2π is a circle, the arcs go backwards from what I'd expect).

Drawing Text

To draw text on a canvas, you first need to define a font size and family (pen.font="30px Arial";) and then you call either stroketext or filltext with the text and the x,y of the bottom left of the text. Below is what both methods look like

Gradients

You can make gradients in HTML using the createLinearGradient function, which takes the x,y of the beginning and end of the gradient (usually something like 0,0,x,0). Color stops need to be added with addColorStop, which 1)defines the colors of the gradient, and 2) makes it visable when used in a fill like in fillRect by editing the fillStyle.

Additionally, a circular gradient can be made with createRadialGradient by defining the x,y and raidus of the starting and ending circles (usually the x and y are similar, but r1 is small and r2 is large(r)).

Images in Canvas

An image can be drawn in a canvas by getting the id in the same way you get the id of the canvas, and by utilizing the drawImage function (pen.drawImage(img,x,y)).

SVG

An SVG element contains SVG graphics, XML-based vector graphics that, since they are vector-based, they don't lose quality when they are resized.

SVG Examples

Included below are some code snippets of SVG elements and what they produce on the screen.

Create a circle:

        
            <svg width="80" height="80">
                <circle cx="40" cy="40" r="20" stroke="green" stroke-width="2" fill="red" />
            </svg>

Create a rectangle:

        
            <svg width="80" height="80">
                <rect cx="10" cy="10" width="60" height="30" stroke="black" stroke-width="4" fill="green" />
            </svg>

Create a fancy square:

        
            <svg width="80" height="80">
                <rect x="10" y="10" rx = "10" ry="10" width="80" height="80" 
                style="fill: green; stroke: black; stroke-width: 4; opacity: 0.5;"/>
            </svg>

Create a star:

        
            <svg width="120" height="120">
                <polygon points="40,10 20,60 70,30 10,30 60,60"
                style="fill: cadetblue; stroke: black; stroke-width: 4; fill-rule: evenodd;"/>
            </svg>

Create an ellipse with gradient and text:

        
            <svg width="500" height="130">
                <defs>
                    <linearGradient id="gradSVG">
                        <stop offset="0%" stop-color="orange"/>
                        <stop offset="100%" stop-color="purple"/>
                    </linearGradient>
                </defs>
                <ellipse cx="100" cy="70" rx="85" ry="55" fill="url(#gradSVG)"/>
                <text fill="#ffffff" font-size="45" font-family="Verdana" x="50" y="86">Text</text>
            </svg>

More Basic CSS

Selectors

In a stylesheet, a selector controls what elements are being modified by your CSS code. There are many different kinds and they will be listed here:

Name, Class, Id
The code edits all elements with the same name/class/id as the selector. i.e editing all paragraphs with p
*
* is the universal selector, it edits all elements on a webpage
Grouping
You can group multiple selectors together to modify them in the same way i.e h1,h2,h3 {}
Descendant Combinator (a b)
Edits all of element b that are descendants of elements a (div p {} edits all paragraphs in div elements)
Child Combinator (a > b)
Edits all of element b that are children of element a (div > p)
Next Sibling Combinator (a + b)
Edits all of element b that is next to element a (div + p edits all paragraphs immedietly following a div)
Subsequent-Sibling Combinator (a ~ b)
Edits all of element b that appear after an element a (div ~ p edits all paragraphs after the first div on the page)
First-child
A psudo-class that selects the first-child of a specifed element (div p:first-child would select the first paragraph in every div)
Lang
A psudo-class that allows you to define different rules for websites in different languages.
Focus
A psudo-class that describes when a text-box is clicked on and your in the state where you can type in it.
First-line
A pseudo-element that indicates the first line of a paragraph
First-letter
Like first-line but for the first letter
Before
A pseudo-element that indicates to do something before the specified element (like put a picture before each header)
After
Like before, but for after each specified element
Marker
Selects the markers of list elements (bullet points, number in ordered lists etc.)
Selection
Denotes the area that is selected (highlighted) by the user.

Pseudo-classes

Pseudo-classes are states an object can be in, but aren't always, like a link being visited or not, or an element being hovered over. They can be combined with default selectors to make more specific modifications. Examples include: a:visited (a link that has been visited over), div.highlight:hover (a div in the highlight class that is being hovered over)

Pseudo-elements

Pseudo-elements are used to style specific parts of an element, such as the first line of a paragraph. They can be combined with selectors like pseudo-classes. Examples include: ::selection (any text that the user highlights), h1::after (place something after each h1 header), etc.

Attribute Selectors

Attribute selectors select elements by their attributes, and there are various modifiers to attribute selectors. A list of examples is kept below.

  • a[target] selects all links with a target attribute
  • a[target="_blank"] selects all links with a target attribute of blank
  • [title~="flower"] selects all elements with an title that contains the word "flower" (wouldn't select "flowery")
  • a[class|="top"] selects all elements with class of either equal to "top" or "top-..."
  • a[class^="top"] selects all elements with a class that starts with/is "top"
  • a[class$="test"] selects all elements with a ends with/is "test"
  • a[class*="text"] selects all elements with a class that contains/is "text"

Comments

Comments in CSS are not the same as comments in HTML, which is a bit annoying. For CSS though, the comments are simple just /*this*/. Even in an HTML file, in the style element, you use the CSS comments.

Text-Decoration

Used to define things like underline, overline and/or line-through text

Text-decoration-color defines the color of these decoration lines, and text-decoration-style tells it what kind of line to use, like the ones for borders, text-decoration-thickness defines how thick the underline is (auto, x%, #px), but these can all be defined in the text-decoration property.

Note: all links are underlined by default, so you can do text-decoration: none; to avoid that if you want to.

Text-transformation

You can use the text-transformation property to automatically make all text in an element lowercase, uppercase, or capitalize(d).

Text Spacing Properties

There are a variety of CSS porperties that affect the spacing of text in an element. Below is a list of them.

text-indent
Adds an indent to the first line of text, like a tab.
letter-spacing
Edits the space between characters.
line-height
Edits the space between lines of text in an element.
word-spacing
Edits the space between words in text.
white-space
Defines how white space is handled in text; nowrap disables text wrapping.

Text-shadows

The text-shadow property takes two values to create a horizonal and vertical shadow effect on the text.

You can also edit the color of the shadow and add a blur effect (of a px size) to the shadow

You can also do the same with the box elements, like divs, by using the box-shadow property instead.

List Styling

The list style property is a shorthand property that contains multiple elements of styling lists; list-style-type defines what the bullet points in a list look like, list-style-image replaces the bullet points in a list with an image (will use list-style-type if image can't be loaded), list-style-position determines if the bullet points are included inside or outside of the list element (determines if it'd be inside of borders or not).

Overflow

Some elements, like tables, can be given scroll bars, so they can be longer horizonally than the screen using the style property overflow-x:auto;. Overflow itself has a couple possible values that will affect how overflow values display. Visable means the extra data falls outside of the element and is displayed below (and is default). Hidden removes all data that is falls outside of the predefined size of the element (like a div). Scroll creates a scroll bar for vertical and horizonal access, and auto creates a scroll bar, if overflow data exists.

Max-width

When dealing with small screens, using the width property (with px values) can lead to an element being too big for the screen and having to scroll horizonally to read it. To prevent this, you can replace the width with max-width, which will make the element either the width of screen or the width you set, whatever is smaller.

Position

The position property changes where the element is placed on the screen depening on various factors.

Position: static is the default behavior of elements.

Position: relative keeps the default behavior, except the left, right, top, and bottom properties can be altered to move the element said amount from its default position.

Position: fixed keeps the element in its same position reguardless of where you are on the page, you can use the left, right, top, and bottom elements to position it.

Position: absolute works like fixed, but for the nearest anchor element, like a div.

Position: sticky is displayed like a releative element, until a certain scroll position, from which it says in that position on the page, like a fixed element.

Z-index

The z-index value of items determines in what order they will be placed on the screen, that value can be positive or negative and the elements are placed from lowest to highest z-index. This would be good for making a small background image around a piece of text, for example. Without z-indexing, the elements are placed from top to bottom in the HTML code.

Clear and Clearfix

The clear property effects how elements interact with other elements with the float property. By default (clear: none), elements are not pushed below elements that are floated left/right, but clear can be set to left/right/both to be pushed below left/right/either float elemenets. Sometimes, you want to put an element with float in a div, but if the element is too big, it will overflow out of the div. Try using this code wth the div. 

    
    .clearfix::after {
        content: "";
        clear: both;
        display: table;
        }

Sprites

To save on server requests for multiple images, multiple small images can be combined into a sprite sheet. You can then use left and width values to select only the sprite you want to use.

Example: #img 1 {left: 0px; width:50px; ...} #img 2 {left: 50px; width: 50px; ...}.

Counters

In CSS, you can create counters that increment by declaring various elements in order to do things like number sections in an article. x{Counter-reset: name} declares a counter called name that resets whenever the x element the counter-reset property is declared. x{Counter-increment: name} add one to the counter called name whenever the x element is declared. In the content property, the counter(name) function can be called to get the value of the counter 'name'.

Specificity

If multiple selectors conflict in how they modify a specific element, the one with the most specifity will win out. The hierarcy of that is as follows:

inline elements ⇒ id selector → (pseudo) class selector → attribute selectors (type="text") → (pseudo) element selectors.

The !important flag can be added to a porperty to ignore any specifity rules and apply that rule to an element.

Math Functions

There are variety of simple math functions you can do in CSS for various reasons. Listed below are a small selection thereof:

Calc
Performs the calculation in the input (width: calc(100% - 100px))
Max/Min
Selects the biggest/smallest value from a list of values (width: max(50%, 300px))

Backgrounds for Divs

Backgrounds can be added to elements that aren't the body, using all of the properties that are listed in previous sections with the adition of two new ones, background-origin (for images) and background-clip (for colors). Both have three possible values: border-box, padding-box and context-box. Border-box means that the background starts at the edge of the border, padding-box means the background starts at the edge of the padding, and context-box means it starts at the edge of the content (based on the box model).

Color Keywords

When inputting a color there are a few special keywords that can be used. Currentcolor holds the current value of the color property of the element (so a border could be made the same, for example). Transparent insets rgb(0,0,0,0) for the color of an element, making it invisible, which has its uses.

Linear-Gradient(s)

Beyond other options you can create gradients in CSS. Linear gradients can be made with the linear-gradient(direction(optional), color 1, color 2, ...) command (here its in background-image). Angle can either be an angle in degrees or keywords such as to right, to top left, or to bottom right.

Radial-Gradient(s)

Radial gradients come from the center and go outward, you can define how much space is between each color step using percents after each color (example: red 5%, blue 15%, green 50%). You can also use the keywords closest-side, farthest-side, closest-corner, or farthest-corner to affect the size of the gradient, as well as add circle to change the shape of the gradient to a circle from an elpise.

Conic-Gradient(s)

Conic Gradients look kinda like pie charts. They default each color to be evenly spaced apart, but you can give degrees in (x)deg format to space out each color break. If you add border-radius equals 50% to the radius, it will look exactly like a pie chart. You can also move the center of the conic gradient with at x% y%.

Repeating Gradients

For any of the gradient styles, you can add repeating- to the function to make it repeat the same colors you provide over and over again. It just needs a % at the end of the last color to show how much space each repetition should take up (i.e red ,blue 20%).

Text-Overflow

When overflow text is hidden from an element (using overflow: hidden;), the text-overflow property determines how to display that text is hidden. Clip cuts off the text at the length of the textbox, while ellipsis adds a ... to the end of the text to show some is missing.

Word-Wrap and Word-Break

The word-wrap property allows for long words to be broken and wrapped onto the next line if set to break-word.

The word-break property defines some line breaking rules, keep-all will break on hyphen and spaces, while break-all will break on any character.

Writing-Mode

The writing-mode property allows for some text to be written vertically using vertical-rl or vertical-lr.

Alternative Fonts

If you want to use a font that is isn't default on a browser, instead of grabbing it from something like the Google API every single time, you can also download the font file and declare it in your stylesheet. You do this by making a @font-face rule, where you specify the font-family, the name your giving the font, and the src, the link to the font file. You can also add things like font-weight:bold if you are adding a bold font or font-style:italic for an italic font.

The Transform Property

The transform property allows you to move/rotate elements in a variety of ways. Listed below are some of the possible functions usable with the transform property

Translate(x,y)
Moves the element x and y pixels on the x and y axis.
Rotate(x)
Rotates the element x degrees clockwise(formatted like 20deg). Can also use negative values for counter-clockwise rotation.
Scale(x,y)
Increase the size of the element by a factor of x horizonally and y vertically.
ScaleX(x)/ScaleY(y)
The same as scale, but only for one dimension.
Skew(x,y)
Skews(kinda tilts) the element by x degrees on the x axis and y degrees on the y axis.
SkewX(x)/SkewY(y)
Like the skew function but only for one axis.
Matrix()
Combines all of the previous transformations into one function in the order matrix(scaleX(), skewY(), skewX(), scaleY(), translateX(), translateY()).
RotateX(x)/RotateY(y)/RotateZ(z)
Rotates the element in 3D space in the specified axis by x degrees.

Transitions

When the property of an element changes, it can be accompanied by a little animation called a transition. It is specified by the transition property and you give 1) the property or properties you want the transition(s) for and 2) how long you want the transition to take (for each property). An example is shown below.

Transition-Timing-Function(s)

The transition-timing-function defines the speed curve of the transition effect. Ease, which is the default, starts slow, then goes faster, and then ends slow. Ease-in and ease-out have a slow start and end respectivly and ease-in-out has both without speeding up as much in the middle. Finally, linear goes a constant speed throughout the transition and cubic-bezier(n,n,n,n) lets you define your own speed function.

Animations

Animations are more involved than transitions, but allow for more control. They start with a @keyframes rule that you give a name and define keyframes. Keyframes are the states that the animation will move between like, in the example below, the element being a blue square and a green rectangle. These are defined in brackets and can either be done with from and to if there are only two states, or by using a percent-way through the animation.

Then, you give define the animation-name property as the name you gave to the keyframes rule and define other animation properties, like animation-duration, how long the animation lasts, animation-iteration-count, how many times an animation is played (infinite for continued playing), animation-timing-function, which defines the timing function like transition-timing-function, animation-direction, the order it plays the keyframes (normal, reverse, alternate, alternate-reverse), animation-delay, which adds a delay before the animation starts, or animation-fill-mode, which applies the first (backwards), last (forwards) or both the first and last (both) keyframe effects to the element when the animation is done.

Filter(s)

The filter property allows for the use of various functions to modify how an element (usually intened for pictures, but you can use them on anything) looks. Below is a list of some of these filter functions.

Blur(Xpx)
Adds a blur effect to an element like this. A bigger value means more blur.
Brightness(x%)
Increases/decreases the brightness of an element tp x percent. 0% means the image is completely black; 100% is the default image.
Contrast(x%)
Increases/decreaes the contrast of an element in the same way as brightness(). 0% makes the element completely gray.
Drop-Shadow(Xpx,Ypx,Zpx,color)
Adds a drop-shadow effect to the element, defined by 2-3 lengths and an optional color field.
Grayscale(x%)
Adds a grayscale effect to the image where 0% or 0 is the normal image and 100% or 1 is completely grayscale.
Hue-rotate(Xdeg)
Applies a hue rotation to an element, by plotting hue values of the image on a circle and moving them along it x degrees (using HSL). Positive numbers increase hue, negative decrease it.
Invert(x%)
Inverts the colors of the image, applied similarly to the grayscale() function.
Opacity(x%)
Effects the opacity of an element; 0% is completely transparent, 100% is default image.
Saturate(x%)
Modifies the saturation of the colors in the image. 0% is desaturated (gray), 100% is default, and 200% is super-saturated.
Sepia(x%)
Adds a sepia filter (warmer, brown/yellow tone) to the image, applied in a similar way to the opacity() function.

Clip-Path Property

The clip-path property allows for an element to be cropped into some simple shapes. Circle(x%) crops the element into a circle with a radius of x% of the element's size. Ellipse(x%, y%) creates an ellipse with an horizonal radius of x% of the element's width and a vertical radius of y% of the element's height. Finally, polygon(x1%, y1%, x2%, y2% ...) lets you define a shape by define x,y points using the % of the elements size.

These functions can also be used with the shape-outside property, which effects how text wraps around an element (you'd might want it to be the same shape as the element it wraps around).

Object-fit Property

The object-fit property effects how an element behaves when placed in an element too small to contain it. It can be fill (default), contain, cover, none, or scale-down. Fill resizes the image to the given dimensions, but this can destroy the aspect ratio. Cover crops the image to fit, which keeps resolution and aspect ratio, but content can be lost. Contain resizes the image to fit in the element, while keeping the aspect ratio, but this won't (usually) use all of the avaliable space. None doesn't resize the image (acts like cover if it don't fit), and scale-down uses the smallest of contain or none.

The object-position property can be used with object-fit: cover to choose what content is cropped out. It takes two % values that choose the center of the image and the crop is done from there.

Mask-image

A mask is an image (png or svg), linear gradient, or SVG drawing that defines an area in what parts of an image to show and which parts to hide (an example is below). A mask is defined in the div the image is in with the mask-image property. Mask-repeat: no-repeat insures the images is only used once instead of covering the size of the base image. Other properties for the mask exist, like mask-size, mask-origin, or mask-position.

Div Columns

A div can be broken into a series of columns and different elements can be placed in the different columns. Column-count determines how many columns are in the div, while column-span determines how many columns an element takes up. Column-style defines the border between columns like the border property does. Column-width and column-gap define the width of the columns and the padding between columns respectivly. Finally, column-fill determines what color to fill the columns with.

Resize

The resize property lets you resize an element horizonal(ly), vertical(ly), or both depending on its value. Make sure to think about overflow when you enable resize.

Note: Some elements (like textarea) have resize on by default, so you can set resize to none to disable that behavior.

Variables

In CSS, you can use variables for all sorts of stuff with the var function. var(--white, #ffffff) would insert the value of the white variable, except if that variable doesn't exist, instead it would insert #ffffff (this second part is optional). To define a variable globally, you can put it in a :root rule as --name: value. You can also define a variable as --name value in any element, but it will only be usable for that element; this can also be used to temporarly override a global variable.

Additionally, Javascript can be used to change CSS variables, specically r = document.querySelector(':root'); will grab all of the variables and r.style.setProperty(--name, newValue); will change the value of the variable. Finally, variables can be changed conditionally in CSS using things like media queries. In them you can use things like minwidth and maxwidth to set new value for variables on different devices.

@Property

To add some functionality, you can create your own properties with the @property rule. Here is an example of how one is defined.

        
            @property --name {
                syntax: "<datatype>";
                inherits: true;
                initial-value: value;
            }

An example of how this can be used to add new functionality, is to create two new color properties that get used for a gradient and which can be changed with an animation, so you can animate a gradient, which you couldn't without doing this.

Flexbox Design

Flexbox design is a design principal where the webpage is divs within divs for sections of the website. When setting the display property to flex of the container, it enables the use of a variety of flex-related properties for both flex containers and items.

For the container, flex-direction defines how items in the container are displayed: row (left to right), column (top to bottom), reverse-row, or reverse-column. The flex-wrap property determines if the items in the container will wrap to a new line if the div is too big for the screen or shrink to fit the screen (nowrap). The justify-content property is used to align items horizonally. Center centers the elements while flex-start and flex-end place the elements at the start and end of the container respectivly. Space-around, space-between, and space-evenly all equally space out the items across the container with the space between them organized slightly differently. Align-items does the same, but vertically. Center and flex-start/end can be used the same way, but stretch/normal and baseline are also options. Stretch makes the items as long as the container, and baseline aligns all of the elements on the baseline, which is like the center but slightly different (this works better (maybe) with different sized items). Finally, align-content works about the same as align-items but it aligns the flex lines instead of the items in the container (idk the difference).

For the items in the flexbox, the order property defines what order the items in the div are displayed (defaults to the order defined), while the flex-grow and flex-shrink properties define how much a specific item will grow/shrink when the screen is resized compared to other items in the container. Flex-basic defines the base length of the item, and flex shorthand for both that and flex-grow/shrink. Finally, align-self chooses an alignment method for the item, overriding the method chosen for the container.

Responsive Web Design

Note: A lot of this is seen elsewhere throughout this, but its probably a good thing to repeat it here.

Viewport

At the beginning of an HTML document, you can create a meta element with the below information to define the viewport of the device. This allows you to get the size of the device you are using, so elements can be resized and rearranged according to responsive design. This includes when sizes are listed in percents, among (many) other things.

<meta name="viewport" content="width=device-width, initial-scale=1.0">

Making the Grid

Probably the easiest way to place elements on a page is to set them up in a grid. This is done by defining the width of elements in 100/x% where x is the number of columns you want in your grid and making them all float left (i.e 100/3 ⇛ 33.33% for 3 columns). Also add box-sizing: border-box to all elements so borders/padding don't mess up the width. Finally, you'd put any element(s) you want in a row into one div and make sure to enable clear:both to ensure that it doesn't break other elements. You can do this by defining classes for both columns that define things like width and float and row that define things like the clear. They could look like the following. 

        
            .col-1 {width: 25%}
            .col-2 {width: 50%}
            ...

            [class*="col-"] {
                float: left;
                [insert style]
            }

            .row::after {
                content: "";
                clear: both;
                display: table;
            }

Media Queries

A media query asks a device about its display and sends this information to the webpage; it can then use this to change various CSS properties about the webpage. A popular example of a media query with grid-based design would be to decrease the number of columns for smaller devices, by making the width for all column classes 100%. Alternativly, you can design for mobile first, and make the breakpoint for large devices to split into more columns, which will make your webpage load faster on mobile devices. A list of common breakpoint for device sizes is shown below.

  • 600 px: extra small devices (phones)
  • 768 px: Larger (newer) Phones
  • 992px: Tablets
  • 1200px: Older Laptops and PCs (below 720p)
  • (orientation: landscape): not a breakpoint, but will tell you if the device is a phone / vertical monitor.

Notes on Images

  • If width is set to a percentage and height is auto, it will scale automatically, while keeping its native aspect ratio.
  • If max-width is used instead of width, the image will scale down as needed, but won't get any larger than its base size.
  • Note: both of these tips also apply to videos native on the page (not API).
  • For background images, if background-size is contain, the image will behave more similar to a img element (but still be in the background)
  • If background-size is 100% 100%, it will stretch to fill the area, will if it's set to cover, it will fill the area, but keep the ascept ratio, so parts of the image will be cropped out.
  • Remember that you can use media queries or the picture element to change photos used for different sized screens.

Grid Properties

Alternativly to the method mentioned earlier, you can make a display: grid container in the same way as the flexbox containers. This allows you to use certain grid properties that can be used to define rows and columns in a more streamlined way than what was described above. The grid-template-columns and rows properties define how many rows and columns are going to be in an element. You can give a specfic size, or use auto for each column to the width evenly between columns. Gap can define the padding between columns and rows (2 values), while justify-content and allign-content are used to horizonally and vertically align the content in the grid. Possible values for those two values are spac-evenly, space-around, space-between, and start and end. These are defined better with the flexbox explanation.

The following properties go on the items in the grid rather than the grid container. Grid-column and grid-row define what column/row to place and element in and what column/row it starts at (1 / 3) or how many columns/row it takes up (1 / span 2). Alternativly, you can make a grid-template-area in the container that defines named areas that can then be called with the grid-area property.
