Lists

HTML has three kinds of lists: unordered (ul), ordered(ol), and description lists (dl). The first two are filled item list items (li), while description lists contain tags(dt) that each contain descriptions (dd). List can be nested and also made horizonally by using float (good for navigation bars).

Below is a list of parameters to edit lists:

style: list-style-type (ul)
Defines what to mark each list item with (bullet point, dash, etc.)
type (ol)
Defines what to use to count entries in list (numbers, letters, roman numerals)
start (ol)
Sets starting value for items in list (default 1).

Span

The span tag can inserted inline into text to modify part of the text, mostly for style purposes.

Div

Div is like span, but defines lines of text. Used to group elements together, like for a text box (not the typing in kind). There are lots of ways to put them together in ways that look nice, but that's a later problem.

Classes

Like in anything else, classes define a set to style attributes that can quickly be assigned to an object. This is done by adding class="[name]" inside the starting tag. To define a class, you put the class name with a period (.name) in the stylesheet and define like any other tag. An object can have multiple classes, just seperate with a space. With JavaScript, you can also perform actions on all elements of a certain class by using getElementsByClassName().

Ids

You can give any element an id by adding id='[name]' to its starting tag. This can be used to create a link to it (like a bookmark) or other JavaScript things with getElementById().

Iframes

An iframe is a webpage displayed in a webpage. It is defined like (iframe src='page_location' title='name'). Style can be used to change the window size, border style etc. You can also target an iframe with a link if you set its target to the 'name' of the frame.

JavaScript

JavaScript is used to add fancy functionality to websites. It is commonly used in script elements. For example, JavaScript is used below to display the current time.

Filepaths

Like in Linux, HTML supports relative file paths. You can define the full filepath, or just the filename if the file is in the same folder. Or you can use ../filename if the file is one folder back etc.

Meta Attributes

The meta element goes in the head and help define various metadata fields like keywords for search engines, character set used (UTF-8), and viewport setting for better support for more devices.

Base

The base element defines a base URL from which all urls in the webiste are derived from, to make relative file paths easier.

Semantic Elements

HTML code is often split up into semantic elements, like section, header, article, etc., to make it easier to read, and allows for some reuse. Below is a list of semantic elements and their uses.

Section
A thematic grouping of content, typically with a heading. Basically what all of these definitions are. Chapters, contact info, etc.
Article
A piece of content that makes sense, and could be distributed, on its own. Examples include blog posts, user comments, or news articles.
Sections and Articles
Note: Sometimes you will find sections nested in articles, while other people will nest articles in sections. You can do both, it just depends what you feel like doing.
Header
A header contains some introductory information, like a logo and name, and/or some navigation links. Headers go at the top of the page.
Footer
Footers go at the bottom of a page and can contain various information like authorship and copyright information, sitemaps, back to top links, etc.
Nav
A nav contains navigations links, not too much more to say.
Aside
An aside contains content vaguely related to the content its near. It can go below it or in some kind of sidebar.
Figure and Figcaption
A figure is self-contained content, like a picture or a chart, and a figcaption give the figure a caption.

Forms and User Input

The Form Element

The form element is needed to make a form in HTML. This usually contains a variety of input elements.

Inputs

Inputs are for, shockingly, taking in user inputs. They take a id, name, and type. The name and id are used for other things, but the type defines what kind of input you are making. HTML has many options from text boxes to color pickers, to sliders.

Labels

Most form elements will use a label to describe what the input is asking for. It is defined like <label for='name'>[insert label here]</label>

The Action Attribute

The form element can have an action attribute that defines an action to be performed when the form is submitted. Usually this involves sending the form data to a file on the server.

The Target Attribute

Like links, forms can have a target attribute which determines where to display the responce to the form (see that section for details).

Post vs Get

The form element can contain a method attribute that determines what protocal to use to send the form data. The two main options are post and get. Get is the default, allows for a page to be bookmarked, and is easer, BUT has a size limit of 2048 characters. Post doesn't have this limit and send the data more securely, so sensitive information should always be sent with post rather than get.

Autocomplete

If you turn on autocomplete (autocomplete="on"), the form will have autocomplete based on what the user has inputted before. Not too complicated.

Novalidate

If you turn on novalidate (novalidate), the form will not attempt to do any kind of validation before it submits the form. This is useful if, for example, you wanted a user to be able to save their progress mid-form.

Select

The select element creates a drop-down list. It is filled with options (<option value="name">Name</option>). The list needs an id and a name, and can also have a size, which defines how many values are visable, and multiple, which flags if multiple options can be selected.

Textarea

The textarea element defines an textbox input. It can be size either by using the rows and columns attributes (represents number of characters), or with width and height through style.

Button

A button is set up with (button onclick="something"). For example if onclick='document.location=link', the user will be sent to a different page.

Fieldset

A fieldset is a group of elements in a form. It can include a legend element to give it a title, like figcaption does for figures.

Datalist

A datalist element is a textbox with predefined options that can be selected. The datalist has an id and contains options with values. To select the datalist, an input needs to be created with a list attribute equal to the id of the datalist.

Output

The output element contains the result of some calculation/function performed, like one by a script.

HTML Canvas and SVG

Canvas

A canvas element is a container in which to draw graphics on the fly with JavaScript. An example of how to declare a canvas is as follows: <canvas id='name' width='x' height ='y' style='border:1px solid #000000;'>.

Drawing on a Canvas

To draw on the canvas, Javascript code has to be written. The following code, for example, would draw a line diagionally across the canvas.

        
            var canvas=document.getElementById('myCanvas');
            var pen=canvas.getContext('2d');
            pen.moveto(0,0);
            pen.lineto(100,100);
            pen.stroke();

Moveto and Lineto

In the canvas, the moveTo method moves the pen object to a specific point, and the lineTo method makes a line from where the pen is to a point specific point.

BeginPath, Arc, and Stroke

In the canvas, the beginPath method is used to reset/set a path for the pen object, and arc defines a arc to be drawn on the canvas that is confirmed with the stroke method. In the arc method, the following values are taken: pen.arc(x,y,radius,start angle, end angle) (the angles are in radians, 0 to 2π is a circle, the arcs go backwards from what I'd expect).

Drawing Text

Gradients

You can make gradients in HTML using the createLinearGradient function, which takes the x,y of the beginning and end of the gradient (usually something like 0,0,x,0). Color stops need to be added with addColorStop, which 1)defines the colors of the gradient, and 2) makes it visable when used in a fill like in fillRect by editing the fillStyle.

Additionally, a circular gradient can be made with createRadialGradient by defining the x,y and raidus of the starting and ending circles (usually the x and y are similar, but r1 is small and r2 is large(r)).

Images in Canvas

An image can be drawn in a canvas by getting the id in the same way you get the id of the canvas, and by utilizing the drawImage function (pen.drawImage(img,x,y)).

SVG

An SVG element contains SVG graphics, XML-based vector graphics that, since they are vector-based, they don't lose quality when they are resized.

SVG Examples

Included below are some code snippets of SVG elements and what they produce on the screen.

Create a circle:

        
            <svg width="80" height="80">
                <circle cx="40" cy="40" r="20" stroke="green" stroke-width="2" fill="red" />
            </svg>

Create a rectangle:

        
            <svg width="80" height="80">
                <rect cx="10" cy="10" width="60" height="30" stroke="black" stroke-width="4" fill="green" />
            </svg>

Create a fancy square:

        
            <svg width="80" height="80">
                <rect x="10" y="10" rx = "10" ry="10" width="80" height="80" 
                style="fill: green; stroke: black; stroke-width: 4; opacity: 0.5;"/>
            </svg>

Create a star:

        
            <svg width="120" height="120">
                <polygon points="40,10 20,60 70,30 10,30 60,60"
                style="fill: cadetblue; stroke: black; stroke-width: 4; fill-rule: evenodd;"/>
            </svg>

Create an ellipse with gradient and text:

        
            <svg width="500" height="130">
                <defs>
                    <linearGradient id="gradSVG">
                        <stop offset="0%" stop-color="orange"/>
                        <stop offset="100%" stop-color="purple"/>
                    </linearGradient>
                </defs>
                <ellipse cx="100" cy="70" rx="85" ry="55" fill="url(#gradSVG)"/>
                <text fill="#ffffff" font-size="45" font-family="Verdana" x="50" y="86">Text</text>
            </svg>

Working with Media Files

Video Element

The video element works similarly to the picture element, where a width and height is defined and various source elements are placed in the video element to help with various devices. The controls attribute can be added to the video element to enable controls like play, pause, volume etc. Different sources would be various file formats, like mp4. The autoplay attribute makes the video start automatically (Chromium usually only allows muted autoplay). The muted tag would make the video start muted.

Audio Element

The audio element works the same way as the video element, and has a lot of the same attributes: autoplay, muted, controls, etc.

Object and Embed Elements

Both the objec and embed elements were designed for use with plug-ins, like Flash, or Java applets, but these are mostly unsupported anymore. Beyond that, they are both able to be used to embed an object, like a HTML file or an image, in a HTML file. Here are the formats for these elements:

<object data='file name'> </object>

<embed src='file name'>

Playing Youtube Videos

If you don't want to go through the hassle of dealing with video formats, you can instead insert videos into your website using Youtube. To do this you make an iframe with its src equal to a youtube link ("https://www.youtube.com/embed/tgbNymZ7vqY") with an embed tag. You can add arguements (after a ? and seperated by &s) to the link to enable certain properties:

autoplay=1 turns on autoplay, mute=1 mutes the video by default. playlist='videoID' along with loop=1 makes the video loop forever. controls=0 disables the player controls.

Working with APIs

getCurrentPosition

The navigator.geolocation.getCurrentPosition(x) method is used to retrieve the user's position and send it to the method x. That position contains latitude, longitude, accuracy, altitude, and altitude, altitudeAccuracy, heading, speed, and timestamp if avaliable in name.coords.

getCurrentPosition can also take a second parameter, which is a function to handel errors. You could have a general responce or a different one for different kinds of errors. error.PERMISSION_DENIED, the user turned off location permissions for the website, error.POSITION_UNAVALIABLE, the location data is unavalible, error.TIMEOUT, location request timed out, or error.UNKNOWN_ERROR, misc. unknown error.

watchPosition

The watchPosition method is similar to getCurrentPosition except it runs constantly and returns updated positions (useful for stuff like GPS). The clearWatch method turns this off.

Drag and Drop

To enable drag and drop on an element takes a lot of steps, but it's not paticularly complicated. Here are a list of the things that need to be done:

Make an element draggable
Add draggable="true" to an element to make it draggable.
Define dragging behavior
A function needs to be written that defines what to do when dragging starts and placed in the element as ondragstart="functionName(event)".
Define droppable locations
By default, a dragged element can't be dropped anywhere, so you need to define behavior in a function for elements to accept dragged elements (at least event.preventDefault()) and put it in ondragover="functionName(event)".
Define drop behavior
You need to preventDefault() again and define what should happen when an element is dropped on another element in a function and place it in ondrop="functionName(event)" of the object being dropped on.

Web Storage

Websites like to store user data to make navigation more convient, like remembering your account so you don't have to log in every time you visit the page. This can be done with cookies, but webstorage allows for much more data to be saved, but is not supported by some older browsers. Before you use it you want to check if web storeage is supported (if (typeof(Storage) !== "undefined")).

LocalStorage

LocalStorage is web storage that has no experation date (it stays when the tab is closed). You can use localStorage.setItem("key", "value"), localStorage.getItem("key"), localStorage.removeItem("key") to manipulate the local storage. It can also keep some special values, like localStorage.clickcount.

SessionStorage

SessionStorage works in the exact same way as localStorage, except sessionStorage is only for the session, meaning that it resets when the browser is closed.

Web Workers

A web worker is a JavaScript that runs in the background of a webpage without affecting performance of the webpage. Before you use it you want to check if web workers are supported (if (typeof(Worker) !== "undefined")). You create a worker with w = new Worker("JavaScript file"), w.terminate() to delete a worker, and w = undefined to reset a worker. It is important the script in the worker utilizes messages, or it won't communicate with the page; w.onmessage determines what will happen when the worker recieves an input for the webpage.

Server-Sent Events (SSE) API

SSEs help web pages get updates automatically from a server, like when a new email shows up automatically when its sent without you having to refresh the page. To create a SSE source, you do new EventSource('file name'), and they communicate with messages and event.onmessage, like web workers. Before you use it you want to check if SSEs are supported (if (typeof(EventSource) !== "undefined")).
