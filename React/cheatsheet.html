React Cheatsheet

A little guide about how React works

React Basic

Below are a list of basic syntax and formatting of how a React file works.

  • In React, you work by defining functions that contain componet(s), you define one function with export default to be the main function to be ran to load the page.
  • A function, by default, can return only one HTML element, but a block surrounded by <> allows for a function to return a set of elements, or other functions can be called by a function that will return their own elements.
  • To call a function, you do <FunctionName field = {value} />. Note that functions have to start with an uppercase (Ex. Profile not profile).
  • To write cleaner code, you can import componets (functions) from other files like "import FunctionName from 'file_location';" (Ex. import Gallery from "./Gallery.js";).
  • Note with importing functions, you can only import functions with the export keyword, and if its not the main function it needs to be in { these }.
  • There are some small extra rules for JSX (the HTML in React), like every tag needing to be closed, but you can also you a converter like this to do it for you.

Variable Basics

With React, you can declare a variable in ways like Javascript (let name, etc.) or you can declare a variable like "const [winner, setWinner] = useState(null);". Note that this requires useState to be imported from React. This example creates a variable winner with value null, and a function setWinner(value) that changes the value of winner to value. To use a variable in an HTML element, it has to go in {brackets}. You also have to do this for any non-string value (tutorials lead me to believe that {100} and 100 would lead to differnet values).

You can also declare an object inline in an HTML element, which is normally inclosed in {brackets}, so you'd need to use double {{brackets}}, this is useful for changing the CSS of an element.

Functions

Functions in React can serve two purposes, they either define elements to display on the page like HTML or providing some kind of functionality, and the later functions are nested in the former. Element inputs define their inputs like FunctionName({ name, size }) and would be called like <FunctionName name = {name} size = {size} />. Interactive functions, on the other hand, are called and defined like Javascript functions. They are defined inside of the element function in which they need to be used (like defining a onclick function for a element). Note that with using functions in the inputs to a element function, you might need to define it like this {() => functionName(name, size)} cause it can cause an infinite loop if you don't sometimes (I don't 100% remember).

Better Lists

To make a basic list, you would wrap li elements in a ul tag, but with Javascript/React you can make them a bit more reactive (haha). You can make an array of objects and then use a map function to make a list of li elements using data.map(data => <li>{dataField}</li>). Additionally, you could use the filter function before you map to only keep elements in data that meet a specific criteria (data.filter(data => data.language === 'English')). This allows you to make a list that can update dymanically. This is something that you could also do with incoming data from something like a SQL query, but you'd probably just do this kind of sorting in the query instead, but you have the option.

Note that to avoid an error (it doesn't break anything, but it's a little annoying), you need to give each of your li element a key. That key can be from some id that's in each object, or a randomly generated number.
